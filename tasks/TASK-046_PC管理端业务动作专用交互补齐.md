# TASK-046 PC 管理端业务动作专用交互补齐

## 目标

在 TASK-044 页面补齐基础上，进一步补齐 PC 管理端的专用业务动作交互，避免关键页面只具备列表和新增能力。

## 范围

- 账单管理：批量生成、催缴、作废。
- 退款管理：审批通过、审批拒绝。
- 工单管理：受理、驳回、派单、开始、挂起、恢复、完工、返工、回访、取消、SLA 超时扫描。
- 消息记录：派发待发送、重试失败、单条重试。
- 租赁合同：生效、终止、到期提醒。
- 门禁权限：权限同步。
- 导入批次：错误 CSV 下载。
- 权限种子补齐：消息派发/重试、导入批次列表/详情/错误明细。

## 验收标准

- 通用列表页可根据页面 key 展示对应业务动作。
- 有必填参数的动作必须弹窗收集参数。
- 操作成功后刷新列表或触发下载。
- 权限码与后端 `@RequiresPermission` 保持一致。
- 不接入真实微信支付、生产环境资料或第三方厂商实机联调。
