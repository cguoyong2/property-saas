# 云端人工验收清单

更新时间：2026-06-28

## 1. 验收前准备

| 项 | 要求 |
| --- | --- |
| PC 管理端地址 | https://property.yqej.cn |
| 移动 H5 地址 | https://property-h5.yqej.cn |
| 管理员账号 | `property_admin` |
| 管理员密码 | 由项目负责人单独交付，不写入文档 |
| 测试小区 | 左岸丽川 |
| 测试业主/住户 | 陈X勇 |

验收注意：

- 不要执行真实收款、退款、作废等会改变正式账务状态的操作。
- 如需演示收款，应先确认是否使用演练库或先做数据库备份。
- 微信支付、第三方设备、真实生产资料当前按既定决策暂缓，不纳入本轮验收。

| 业主/住户 | `/base/members` | 陈X勇手机号为 11 位，关联小区、楼栋、单元、房屋 |
| 房屋绑定审核 | `/base/member-bindings` | 可看到移动端/PC 端提交来源，支持审核动作 |

## 3. 收费账务验收

| 模块 | 页面 | 验收点 |
| --- | --- | --- |
| 收费项目 | `/fee/items` | 收费项目为中文显示，编码由系统生成 |
| 收费标准 | `/fee/standards` | 房屋物业费按建筑面积，车位物业费固定金额 |
| 收费绑定 | `/fee/standard-binds` | 房屋标准绑定房屋，车位标准绑定车位 |
| 账单管理 | `/fee/bills` | 自动显示陈X勇房屋物业费 3600 元、车位物业费 360 元 |
| 账单明细 | `/fee/bills` | 显示业主/住户、手机号、房号、账期、应收明细 |
| 收款订单 | `/payment/orders` | 当前正式库应为空 |
| 退款管理 | `/payment/refunds` | 当前正式库应为空；新增时应选择已收款订单 |
| 支付流水 | `/payment/flows` | 当前正式库不应有真实流水 |
| 支付对账 | `/payment/reconcile/exceptions` | 当前无对账异常 |

## 4. 移动 H5 验收

| 页面/接口 | 验收点 |
| --- | --- |
| https://property-h5.yqej.cn | 页面可打开 |
| 首页 | 公共服务入口、公告和服务入口可见 |
| 服务 | 未绑定房屋时可看公共服务；绑定后展示专属服务 |
| 消息 | 可展示公共公告和绑定后通知入口 |
| 我的 | 可查看绑定状态、房屋、家属、账单、车辆等入口 |
| 公共公告接口 | `tenantId=202606280001` 时返回 HTTP 200 |
| 登录接口 | 使用 POST 请求，不用 GET 判断可用性 |

## 5. 不纳入本轮验收的内容

| 内容 | 原因 |
| --- | --- |
| 微信支付真实收款 | 等真实部署和商户资料齐备后接入 |
| 第三方门禁/停车/设备厂商 | 等现场安装设备后对接 |
| 真实生产资料导入 | 等正式上线资料确认后完善 |

## 6. 验收后动作

1. 修改 `property_admin` 密码，并记录到项目方密码保管渠道。
2. 如果需要增加验收人员账号，在 PC 管理端通过系统用户管理新增，不共用管理员账号。
3. 验收通过后做一次云端数据库备份。
4. 如发现问题，先截图记录页面、时间、操作路径，再决定是否回滚或修复。
