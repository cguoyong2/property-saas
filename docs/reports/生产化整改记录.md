# 生产化整改记录

## 1. 生产配置与密钥整改

已完成：

- 新增 `.env.example`，保留本地开发默认值。
- 新增 `.env.prod.example`，生产配置不提供默认密钥或默认密码。
- 新增 `backend/src/main/resources/application-prod.yml`，生产 profile 从环境变量读取数据库、Redis、JWT 和微信支付模式配置。
- 新增 `docker-compose.prod.example.yml`，生产示例 compose 使用 `${VAR:?required}` 强制要求密钥变量。
- 修改 `docker-compose.yml`，本地默认值统一走环境变量，并使用 `sql/seed_demo_data.sql`。
- 修改 PC 登录页，生产构建不再预填演示账号和密码。
- 新增 `scripts/prod-env-check.sh`，上线前检查必需环境变量、JWT 长度、默认占位值和微信支付模式。
- 生产迁移目录已在临时库验证通过：57 张表、0 个演示租户、0 个演示用户、140 条权限菜单。

生产启动前建议执行：

```bash
source .env.prod
./scripts/prod-env-check.sh
```

## 2. 真实微信支付服务商接入方案

已完成：

- 新增 `docs/deploy/微信支付服务商接入方案.md`。
- 明确服务商模式配置字段、回调验收要求、证书/密钥读取边界、Redis nonce 防重放 key 和禁止事项。
- `application-prod.yml` 默认要求 `WECHAT_PAY_MODE=REAL_WECHAT_V3`。

边界说明：

- 当前代码仍保留本地 HMAC 模拟验签，供开发和联调烟测使用。
- 真实微信支付服务商能力需要接入微信平台证书、商户私钥、API v3 key、回调解密和 nonce/timestamp 防重放；这些依赖真实商户资料和证书，不能在工程包中伪造完成。
