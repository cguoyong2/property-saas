# 2026-06-28 云端生产迁移口径修复报告

## 1. 背景

本次根据上线前检查结果，对云端部署模板和生产迁移链路进行修复。目标是把云端部署从旧的“演示 SQL 初始化 + 关闭 Flyway”调整为“生产 profile + Flyway migration-prod 迁移”。

## 2. 发现的问题

| 问题 | 影响 | 处理 |
|---|---|---|
| 云端 compose 挂载 `sql/schema_mysql8.sql` 和 `sql/seed_demo_data.sql` | 容易使用滞后的旧建表脚本，并把演示数据带入云端 | 已移除 Docker entrypoint 初始化 SQL |
| 云端 compose 使用 `SPRING_PROFILES_ACTIVE=dev` 且关闭 Flyway | 与生产迁移规范冲突，无法验证生产迁移链路 | 已改为 `prod` profile，并使用 `DB_URL/DB_USERNAME/DB_PASSWORD` 等生产环境变量 |
| `migration-prod` 缺少 V33-V35 预存款迁移 | 空库生产迁移到 V43 时，`member_prepayment` 表不存在，启动失败 | 已补齐 V33、V34、V35 到生产迁移目录 |
| 现有云端演示库可能非空且无 Flyway 历史 | 直接切换生产 Flyway 可能失败或产生不可控数据风险 | 已新增云端数据库迁移切换方案，要求先备份和判断库状态 |

## 3. 修改范围

| 文件 | 内容 |
|---|---|
| `deploy/cloud/docker-compose.property-saas.yml` | 改为生产 profile、生产环境变量、Flyway 迁移口径；移除旧 SQL 初始化挂载 |
| `backend/src/main/resources/db/migration-prod/V33__member_prepayment.sql` | 新增预存款表 |
| `backend/src/main/resources/db/migration-prod/V34__prepayment_permissions.sql` | 新增预存款权限菜单 |
| `backend/src/main/resources/db/migration-prod/V35__prepayment_usage.sql` | 新增预存款抵扣明细表 |
| `docs/deploy/云端数据库迁移切换方案.md` | 新增云端演示库到生产 Flyway 的切换步骤和回滚要求 |
| `docs/deploy/生产部署预案与回滚方案.md` | 增加云端迁移切换前置要求 |
| `docs/deploy/预生产演练执行清单.md` | 增加云端库状态检查步骤 |
| `docs/deploy/数据库初始化与迁移规范.md` | 补充云端 compose 使用生产 Flyway 口径 |

## 5. 后续要求

1. 正式切换云端环境前，必须先按 `docs/deploy/云端数据库迁移切换方案.md` 备份数据库并判断库状态。
2. 如果现有云端库是旧演示库且没有 `flyway_schema_history`，不得直接启动生产 Flyway。
3. 微信支付真实资料、生产资料、第三方设备厂商资料仍按项目方确认暂缓，不作为本次修复范围。
