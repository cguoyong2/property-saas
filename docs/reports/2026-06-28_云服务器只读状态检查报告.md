## 1. 检查目标

按上线前建议，对现有云服务器数据库做只读状态检查，判断当前库属于以下哪一类：

1. 空库，可直接按生产 Flyway 迁移。

3. 非空库但没有 `flyway_schema_history`，不得直接硬切生产迁移。
4. 旧演示库混入生产数据，需要先备份、清理或重建。

## 4. 当前结论

云服务器公网 IP 和端口是可达的，但服务层没有正常响应：

1. 22 端口可连接，但 SSH 服务不返回 banner，无法登录执行只读数据库检查。
2. 443 端口可连接，但 TLS 握手超时，公网 HTTPS 服务当前不可用。
3. 80 端口可连接，但 HTTP 请求无响应。

因此，本次未能进入服务器执行数据库 `flyway_schema_history` 和表结构检查。当前状态更像服务器内的 Nginx/SSH/系统负载/容器运行状态异常，而不是 DNS 解析或安全组完全未放行。

## 6. 建议处理

1. 先登录阿里云控制台查看 ECS 状态、CPU、内存、磁盘、带宽是否异常。
2. 如果控制台显示实例正常，优先通过阿里云 VNC/Workbench 进入服务器执行：

```bash
systemctl status sshd --no-pager
docker ps
docker logs --tail=100 global-edge-nginx
docker logs --tail=100 property-saas-backend
df -h
free -h
uptime
```

3. 如果 SSH 服务卡死但控制台可操作，可重启 `sshd`；如果整机负载异常，应先定位负载来源，不建议直接切换生产 compose。
4. 在完成数据库备份和库状态判断前，不执行云端生产迁移切换。
