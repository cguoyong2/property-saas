## 测试中发现的问题

### P1：本地 Docker 初始化 SQL 滞后于迁移脚本

现象：

- 执行 `./scripts/db-reset-local.sh` 后，`docker-compose.yml` 会导入 `sql/schema_mysql8.sql` 和 `sql/seed_demo_data.sql`。
- 该初始化 SQL 未包含部分新字段，例如 `member_house_bind.relationship`、`allow_work_order`。
- 如果直接用该库启动后端并关闭 Flyway，部分接口会因缺字段失败。
- 如果打开 Flyway，因初始化 SQL 已创建部分后续迁移中的表，会出现重复建表错误。

影响：

- 不影响当前公网环境运行。
- 会影响后续本地一键重置后的联调稳定性。

建议：

- 下一步优先统一本地初始化策略：要么移除 Docker 初始化 SQL，让 Flyway 全量建库；要么把 `sql/schema_mysql8.sql` 同步到 V43 后的完整结构，并调整 Flyway baseline 策略。

### P2：本机 9000 端口被其它项目 MinIO 占用

现象：

- `./scripts/db-reset-local.sh` 启动 MinIO 时，因本机已有 `health-minio` 占用 9000/9001，导致 `property-saas-minio` 启动失败。

影响：

- 不影响本轮核心后端、前端、账务、业务接口测试。
- 如果后续要测试本地文件上传/对象存储，需要处理端口冲突。

建议：

- 将本项目 MinIO 本地端口调整为未占用端口，或在测试前停止其它项目 MinIO。
