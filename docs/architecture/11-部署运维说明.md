# 11-部署运维说明

## 1. 环境划分

| 环境 | 用途 |
|---|---|
| dev | 开发自测 |
| test | 测试人员验证 |
| staging | 预发布联调 |
| prod | 生产环境 |

## 2. Docker Compose 组件

- MySQL 8
- Redis 7
- MinIO
- 后端 API
- 管理端前端
- Nginx

## 3. 备份策略

- 数据库每日全量备份；
- 支付、账单、退款、合同等关键表建议增量备份；
- 对象存储启用版本或回收站；
- 备份文件加密保存；
- 每月至少做一次恢复演练。

## 4. 监控告警

必须监控：

- 登录失败异常；
- 支付回调失败；
- 退款失败；
- 短信发送失败；
- 门禁/停车同步失败；
- 数据库慢查询；
- Redis 连接异常；
- 磁盘空间不足；
- 租户到期；
- 批量账单生成失败。

## 5. 日志保留

| 日志 | 保留建议 |
|---|---|
| 操作日志 | 不少于 180 天 |
| 支付日志 | 长期保留 |
| 账务变更日志 | 长期保留 |
| 接口调用日志 | 不少于 180 天 |
| 登录日志 | 不少于 180 天 |

## 6. 生产上线预案

生产发布前必须完成：

1. 填写 `docs/deploy/生产上线资料收集表.md`。
2. 确认 `docs/deploy/生产部署预案与回滚方案.md` 中的上线准入门禁、发布职责、冒烟验证和回滚步骤。
3. 执行 `scripts/prod-env-check.sh`，确保生产环境变量无默认值、无占位符。
4. 使用 `backend/src/main/resources/db/migration-prod` 执行生产迁移，不执行演示种子数据。
5. 上线前完成数据库全量备份，并记录恢复负责人和备份校验值。
