# 10-第三方接口对接边界

## 1. 微信小程序/公众号

前置条件：

- 小程序 AppID、Secret；
- 服务类目和隐私协议；
- 服务器域名配置；
- 消息模板；
- 客服电话和品牌配置。

## 2. 微信支付

推荐优先级：

1. 服务商/子商户模式；
2. 租户自有商户模式；
3. 平台统一收款再结算不建议作为默认方案。

必须确认：

- 商户号/服务商号；
- API v3 密钥；
- 证书；
- 回调域名；
- 退款权限；
- 分账/结算规则；
- 对账单下载权限。

## 3. 短信平台

对接范围：验证码、催缴、通知、异常提醒。

要求：

- 租户级或平台级签名模板；
- 发送频控；
- 余额或额度控制；
- 失败回执；
- 发送记录。

## 4. 门禁/梯控

必须通过适配层开发：

```text
DeviceAdapter
  - DahuaAccessAdapter
  - HikvisionAccessAdapter
  - GenericHttpAccessAdapter
```

不允许业务模块直接调用具体厂商接口。

所有第三方调用必须写入 `interface_call_log` 或对应业务同步表，至少记录：

- `tenant_id`、`project_id`；
- 厂商编码、接口类型、请求 ID；
- 请求、响应、成功状态、失败原因；
- `retry_count`、`next_retry_at`；
- 调用耗时和创建时间。

前置条件：

- 厂商接口文档；
- 测试环境；
- 账号密码/密钥；
- 设备清单；
- 项目、楼栋、单元、门禁点位映射；
- 人员、人脸、卡号同步规则；
- 失败重试和补偿机制。

## 5. 停车系统

对接范围：车辆档案、月租车、临停缴费、停车记录、优惠券、欠费限制策略。

默认只做接口预留和适配，不承诺所有厂商深度对接。具体厂商需另行确认接口可用性。

停车同步类操作除记录 `interface_call_log` 外，还必须写入 `parking_sync_record`，用于追踪车辆、月租、权限下发等业务同步结果。
