# 02-技术架构说明

## 1. 架构模式

初期采用模块化单体架构，避免过早拆分微服务。系统内部按领域模块划分包结构，后期可按租户、账务、工单、设备适配等领域拆分服务。

## 2. 技术栈

- 后端：Java 17 + Spring Boot 3.x + Spring Security + JWT + MyBatis-Plus + Maven
- 数据库：MySQL 8.x，统一 utf8mb4，金额使用 DECIMAL(12,2)，主键使用 BIGINT
- 缓存：Redis 7.x，所有业务 key 必须带 tenant:{tenantId}: 前缀
- 文件：MinIO / S3 兼容对象存储，路径必须包含 tenant_id/project_id/module
- API 文档：OpenAPI 3.0 + Knife4j/Swagger UI
- 管理端：Vue 3 + TypeScript + Element Plus
- 微信端：uni-app，目标平台为微信小程序
- 部署：Docker Compose 优先，预留 Kubernetes 扩展
- 日志：统一操作日志、接口日志、支付日志、异常日志

```

## 6. 请求链路

1. 前端携带 Token 请求后端。
2. 后端 Security 解析用户身份。
3. TenantContext 确定当前租户。
4. DataScope 确定项目、部门、本人数据范围。
5. Controller 参数校验。
6. Service 执行业务规则。
7. Mapper 自动或显式追加 tenant_id 条件。
8. 高风险操作写审计日志。
9. 返回统一响应结构。

## 8. 错误码原则

| 错误码 | 含义 |
|---|---|
| 0 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或 Token 失效 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 409 | 业务冲突，如重复账单、重复绑定 |
| 429 | 请求过于频繁 |
| 500 | 系统异常 |
| 60001 | 租户不存在或已停用 |
| 60002 | 租户套餐到期或额度不足 |
| 70001 | 支付回调验签失败 |
| 70002 | 支付回调重复 |
| 80001 | 第三方接口调用失败 |
```
