# 09-接口设计说明

## 1. 接口文件

完整接口契约见：`openapi/openapi.yaml`。

## 2. 接口前缀

| 前缀 | 说明 |
|---|---|
| `/api/auth` | 后台登录、菜单、当前用户 |
| `/api/platform` | SaaS 平台运营端 |
| `/api/system` | 租户内系统管理 |
| `/api/base` | 基础档案 |
| `/api/fee` | 收费账务 |
| `/api/payment` | 支付订单、流水、退款、回调 |
| `/api/service` | 公告、工单、投诉 |
| `/api/lease` | 招商租赁、合同 |
| `/api/patrol` | 资产巡检 |
| `/api/device` | 门禁、访客、停车、梯控 |
| `/api/report` | 报表统计 |
| `/api/jobs` | 定时任务与提醒中心 |
| `/api/app` | 微信小程序端 |

## 3. 请求头

| Header | 说明 |
|---|---|
| `Authorization: Bearer <token>` | 登录令牌 |
| `X-Request-Id` | 请求追踪 ID |
| `X-Project-Id` | 当前项目，可选；后端仍需校验权限 |
| `X-House-Id` | 小程序当前房屋，可选；后端仍需校验绑定关系 |

## 4. 不允许的接口设计

- 不允许后端直接相信前端传入的 `tenant_id`。
- 不允许不分页的列表接口。
- 不允许导出接口无权限控制。
- 不允许支付回调只按前端传入参数更新订单。
- 不允许删除账务明细，只能作废或冲正。

## 5. 统一响应

所有业务接口默认返回：

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "traceId": "request-trace-id"
}
```

列表接口的 `data` 必须使用分页结构：

```json
{
  "records": [],
  "pageNo": 1,
  "pageSize": 20,
  "total": 0
}
```

## 6. 错误码分类

| 范围 | 类型 | 示例 |
|---|---|---|
| 400xxx | 参数或业务状态错误 | 参数缺失、状态不允许流转 |
| 401xxx | 登录认证错误 | Token 过期、未登录 |
| 403xxx | 权限或数据范围错误 | 无按钮权限、跨租户、跨项目 |
| 404xxx | 资源不存在 | 房屋不存在、账单不存在 |
| 409xxx | 并发或重复数据 | 重复账单、重复导入 |
| 500xxx | 系统内部错误 | 第三方接口异常、未知异常 |

错误响应必须包含 `traceId`，便于关联接口日志和异常日志。

## 7. 契约优先级

页面清单、任务文档、测试用例中引用的接口必须先在 `openapi/openapi.yaml` 中存在。若页面或任务需要新增接口，应先修订 OpenAPI，再开发后端和前端。
