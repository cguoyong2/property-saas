# 预生产演练执行清单

> 用途：正式上线前，在预生产或空生产环境按最新完成包执行一次可记录、可复盘、可回滚的部署演练。本文不要求当前立即具备真实生产环境，先作为执行模板。

## 1. 演练输入

| 项目 | 内容 | 状态 |
| --- | --- | --- |
| 演练包 | 当前代码工作区最新构建，或重新生成的最新发布包 | 待生成/待确认 |
| 目标环境 | 预生产 / 空生产库 | 待确认 |
| 数据库迁移目录 | `backend/src/main/resources/db/migration-prod` | 已确认 |
| 最新迁移版本 | `V43__member_prepayment_refund_amount.sql`，生产迁移链路已补齐 V33-V35 预存款表和权限迁移 | 已确认 |
| 云端迁移切换口径 | `docs/deploy/云端数据库迁移切换方案.md` | 已确认 |
| 环境变量模板 | `.env.prod.example` | 已确认 |
| 生产环境变量检查脚本 | `scripts/prod-env-check.sh` | 已确认 |
| 冒烟脚本 | `scripts/integration-smoke.sh` | 已确认 |

## 2. 演练步骤

| 步骤 | 操作 | 命令或证据 | 期望结果 | 结果 |
| --- | --- | --- | --- | --- |
| 1 | 解压最新完成包到干净目录 | 解压重新生成的最新发布包 | 包内不包含 `node_modules`、`dist`、`target`、`unpackage`、`data/uploads` | 待执行 |
| 2 | 填写预生产环境变量 | 参考 `.env.prod.example` | 密钥只使用引用或运行时注入 | 待执行 |
| 3 | 执行环境变量检查 | `./scripts/prod-env-check.sh` | 无缺失、无演示密钥 | 待执行 |
| 4 | 准备空库或预生产库备份 | DBA 记录备份路径和校验值 | 可恢复 | 待执行 |
| 5 | 检查云端库状态 | 按 `云端数据库迁移切换方案.md` 查询表和 `flyway_schema_history` | 明确空库、有迁移历史、或非空无迁移历史 | 待执行 |
| 6 | 执行后端测试 | `cd backend && mvn -q test` | 通过 | 待执行 |
| 7 | 构建后端包 | `cd backend && mvn -DskipTests package` | 生成 jar | 待执行 |
| 8 | 执行生产迁移 | 以后端 `prod` profile 启动或 Flyway 执行 `migration-prod` | `flyway_schema_history` 到 v43 | 待执行 |
| 9 | 启动后端服务 | `SPRING_PROFILES_ACTIVE=prod java -jar target/...jar` | `/actuator/health` 可用 | 待执行 |
| 10 | 构建 PC 管理端 | `cd admin-web && npm install --legacy-peer-deps --ignore-scripts && npm run build` | `dist` 生成，JS chunk 无超限警告 | 待执行 |
| 11 | 构建微信小程序 | `cd mobile-uniapp && npm install --legacy-peer-deps --ignore-scripts && npm run build:mp-weixin` | `dist/build/mp-weixin` 生成 | 待执行 |
| 12 | 执行联调烟测 | `API_BASE_URL=<预生产 API> ./scripts/integration-smoke.sh` | 关键租户隔离和支付错误签名拒绝通过 | 待执行 |
| 12.1 | 执行业务动作回归 | `API_BASE_URL=<预生产 API> ./scripts/integration-business-actions.sh` | 工单、消息、导入、业主检索、房屋绑定审核、车位车辆、账单和退款候选检查通过 | 待执行 |
| 12.2 | 执行账务变更回归 | `API_BASE_URL=<预生产 API> RUN_MUTATING_ACCOUNTING_CHECKS=true ./scripts/integration-business-actions.sh` | 仅在可重置预生产环境执行，现金部分收款等改账检查通过 | 待执行 |
| 13 | 人工冒烟 | 按第 3 节执行 | 通过或记录缺陷 | 待执行 |
| 14 | 回滚演练 | 停止新服务、恢复上一版本或恢复库备份 | 回滚路径可执行 | 待执行 |
| 15 | Go/No-Go 评审 | 项目负责人组织签核 | 明确 Go 或 No-Go | 待确认 |
