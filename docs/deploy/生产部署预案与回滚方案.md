## 3. 部署前准备

1. 从最新完成版 zip 解压工程包，确认包内不包含 `node_modules`、`dist`、`target`、`unpackage`。
2. 按 `.env.prod.example` 创建生产环境变量文件，所有密钥项只填写密钥管理系统引用或运行时注入方式。
3. 执行生产环境变量检查：

```bash
./scripts/prod-env-check.sh
```

4. 创建生产数据库，字符集使用 `utf8mb4`，排序规则使用 `utf8mb4_unicode_ci`。
5. 对生产库执行上线前全量备份，记录备份路径和校验值。
6. 替换 `sql/seed_prod_minimal.sql` 中首个平台管理员、角色、套餐等占位符，禁止使用演示账号和演示密码。
7. 确认微信公众平台已配置 request/uploadFile/downloadFile 合法域名。
8. 确认微信支付回调地址和退款回调地址均为公网 HTTPS 地址。

## 4. 标准部署步骤

### 4.1 后端

```bash
cd backend
mvn -DskipTests package
SPRING_PROFILES_ACTIVE=prod java -jar target/property-saas-backend-0.1.0-SNAPSHOT.jar
```

检查项：

| 检查 | 期望 |
|---|---|
| Flyway | 使用 `classpath:db/migration-prod` 并迁移到最新版本 |
| 健康检查 | `/actuator/health` 返回可用 |
| 日志 | 无数据库、Redis、证书、密钥缺失错误 |
| 安全 | 生产日志不打印明文密码、JWT secret、微信私钥 |

### 4.2 PC 管理端

```bash
cd admin-web
npm install
npm run build
```

将 `admin-web/dist` 发布到 Nginx 或对象存储静态站点。上线前确认 API 代理、HTTPS、缓存策略、404 fallback 和安全响应头。Nginx 可参考 `docs/deploy/nginx-security-headers.conf` 配置 CSP、HSTS、X-Frame-Options、Referrer-Policy 等安全头。

### 4.3 微信小程序端

```bash
cd mobile-uniapp
npm install --legacy-peer-deps --ignore-scripts
npm run build:mp-weixin
```

使用微信开发者工具导入 `mobile-uniapp/dist/build/mp-weixin`，先发布体验版并完成验收，再提交审核和正式发布。

## 6. 回滚策略

### 6.1 代码回滚

适用场景：新版本启动失败、接口异常、前端静态资源错误、小程序体验版未正式发布。

步骤：

1. 停止新版本后端实例。
2. 恢复上一个已验证后端 jar 或容器镜像。
3. 恢复上一版 PC 静态资源。
4. 小程序若未正式发布，停止发布流程；若已正式发布，按微信平台规则提交上一稳定版本。
5. 执行冒烟验证，确认登录、账单、支付、退款和审计日志正常。

### 6.2 数据库回滚

适用场景：迁移脚本破坏数据、核心表结构不兼容、账务数据异常。

原则：

1. 优先使用向前修复迁移；只有 P0 数据破坏或核心流程不可用时才恢复备份。
2. 恢复备份前必须冻结写入入口，包括后端服务、异步任务、支付回调入口和第三方回调入口。
3. 恢复后必须核对 `flyway_schema_history`、关键账务表、订单流水和操作日志。

步骤：

1. 宣布进入数据库回滚窗口，停止后端写入服务。
2. 记录当前数据库 binlog 位点或快照。
3. 使用上线前全量备份恢复到独立库，完成数据校验。
4. 切换应用连接到恢复库，或在原库恢复后重新启动服务。
5. 对支付回调和第三方回调做补偿处理，避免重复核销或漏处理。

### 6.3 配置回滚

适用场景：域名、证书、环境变量、Redis、对象存储、微信支付证书配置错误。

步骤：

1. 回退到上一版环境变量和网关配置。
2. 重启受影响服务。
3. 验证健康检查、登录、上传下载、支付回调。
4. 对发布窗口内失败任务进行人工补偿。

## 8. 当前缺口

| 缺口 | 影响 | 处理方式 |
|---|---|---|
| 真实微信支付商户资料未填 | 无法完成真实支付验签、解密、证书轮换验收 | 支付负责人补齐服务商商户号、子商户号、证书、API v3 key 引用 |
| 生产域名和 HTTPS 证书未填 | 无法配置小程序合法域名和支付回调地址 | 运维负责人补齐域名、证书、续期方式 |
| 首个平台管理员信息未填 | 无法执行生产最小初始化 | 项目负责人提供登录名、手机号、BCrypt 哈希 |
| 数据库备份恢复演练未完成 | 数据库回滚不可证明 | DBA 完成恢复演练并记录结果 |
| 第三方设备厂商资料未填 | 设备联调无法进入验收 | 各厂商负责人按验收模板补齐资料 |
