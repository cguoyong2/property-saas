# 云端数据库迁移切换方案

> 用途：把云端部署从“演示 SQL 初始化”切换到“生产 Flyway 迁移”前的执行口径。本文不记录明文密码、私钥或证书内容。

## 1. 目标口径

正式上线和预生产演练统一采用以下策略：

| 项目 | 目标 |
|---|---|
| 后端 Profile | `SPRING_PROFILES_ACTIVE=prod` |
| 数据库迁移目录 | `backend/src/main/resources/db/migration-prod` |
| 数据库初始化 | 由 Flyway 执行生产迁移 |
| 生产迁移连续性 | `migration-prod` 必须包含所有生产结构迁移；演示种子 V2 不进入生产 |
| 演示数据 | 禁止执行 `sql/seed_demo_data.sql` |
| 旧完整建表脚本 | `sql/schema_mysql8.sql` 仅保留为历史审阅文件，不作为运行时初始化入口 |
| 首个平台管理员 | 使用 `sql/seed_prod_minimal.sql` 替换占位符后单独执行或纳入受控初始化流程 |

当前 `deploy/cloud/docker-compose.property-saas.yml` 已调整为生产 profile 和 Flyway 口径，不再挂载 `docker-entrypoint-initdb.d` 初始化 SQL。

## 2. 现有云端环境不得直接硬切

如果云服务器上已经运行过旧版演示部署，数据库可能是以下状态之一：

| 状态 | 判断方式 | 处理方式 |
|---|---|---|
| 空库 | 无业务表或仅有空 schema | 直接使用 `prod` profile 启动，由 Flyway 从 V1 迁移到最新版本 |
| 已有 `flyway_schema_history` | 查询存在迁移历史且最新版本为 43 | 可按标准升级流程启动 |
| 非空库但没有 `flyway_schema_history` | 有业务表但无 Flyway 记录 | 不能直接启动 `prod` profile；需先备份并执行基线处理或重建库 |
| 演示数据混入生产 | 存在演示账号、演示租户、演示账单 | 不建议直接作为生产库；应导出真实数据后重建生产库 |

检查命令示例：

```bash
docker exec property-saas-mysql mysql -u"$DB_USERNAME" -p"$DB_PASSWORD" property_saas -e "show tables like 'flyway_schema_history';"
docker exec property-saas-mysql mysql -u"$DB_USERNAME" -p"$DB_PASSWORD" property_saas -e "select version, success from flyway_schema_history order by installed_rank desc limit 5;"
```

## 3. 推荐切换步骤

1. 停止后端写入入口，包括 PC 管理端、移动端 H5、小程序 API、定时任务和支付回调入口。
2. 对 MySQL 做全量备份，记录备份文件、大小、校验值、负责人和恢复命令。
3. 确认当前库状态：
   - 空库：直接进入第 5 步。
   - 有 Flyway 历史：确认最新版本为 V43，再进入第 5 步。
   - 非空无 Flyway 历史：优先重建预生产库演练；如必须沿用，先由后端负责人和 DBA 对照 V1-V43 做结构差异审查，再决定基线方案。
4. 准备生产最小初始化：
   - 替换 `sql/seed_prod_minimal.sql` 中首个平台管理员、角色、套餐等占位符。
   - 禁止使用演示账号、演示手机号、演示密码和演示租户。
5. 按 `.env.prod.example` 准备环境变量，执行：

```bash
./scripts/prod-env-check.sh
```

真实微信支付资料暂未接入时，可在预生产或内网演示环境临时使用 `WECHAT_PAY_MODE=DEV_SIMULATED`，但该状态不得作为正式公网生产验收结论。

6. 启动云端服务：

```bash
docker compose -f deploy/cloud/docker-compose.property-saas.yml up -d mysql redis
docker compose -f deploy/cloud/docker-compose.property-saas.yml up -d backend
```

7. 确认迁移结果：

```bash
docker exec property-saas-mysql mysql -u"$DB_USERNAME" -p"$DB_PASSWORD" property_saas -e "select version, success from flyway_schema_history order by installed_rank desc limit 1;"
```

期望最新版本为 `43` 且 `success=1`。

8. 启动 PC 和 H5：

```bash
docker compose -f deploy/cloud/docker-compose.property-saas.yml up -d admin-web mobile-h5
```

9. 执行冒烟：

```bash
API_BASE_URL=<云端后端地址> ./scripts/integration-smoke.sh
API_BASE_URL=<云端后端地址> ./scripts/integration-business-actions.sh
```

10. 记录演练结果到 `docs/deploy/预生产演练执行清单.md`。

## 4. 回滚要求

切换失败时按以下优先级回滚：

1. 停止新后端容器，恢复上一版 jar 和上一版 compose 环境变量。
2. 如迁移已经写入数据库但未产生业务写入，优先恢复上线前备份到独立库后切换连接。
3. 如迁移后已有支付、账单、退款、审核等业务写入，必须冻结写入入口，由 DBA 和后端负责人共同确认补偿方案，不得直接覆盖库。
4. 回滚后必须核对登录、租户隔离、账单、收款、退款、文件上传和审计日志。

## 5. 当前暂缓事项

| 事项 | 口径 |
|---|---|
| 微信支付真实商户接入 | 按项目方确认，等真实部署和商户资料齐备后再接入 |
| 生产资料最终填写 | 等正式部署资料齐备后完善 |
| 第三方设备厂商对接 | 等设备安装和厂家资料齐备后对接 |

上述暂缓事项不阻塞内网预生产演练，但会阻塞正式公网生产验收。
