# 生产上线资料责任清单

> 用途：将 `生产上线资料收集表.md` 中分散的待确认、待执行事项按责任角色聚合，便于项目负责人在正式部署前逐项派发和追踪。本文不填写明文密码、私钥、证书或 API key。

## 1. 使用方式

1. 项目负责人在启动正式部署准备会前复制本清单。
2. 每个责任人只填写自己名下条目，密钥类字段只填写密钥管理系统引用或交付位置。
3. P0 条目未全部关闭时，不进入正式发布窗口。
4. 变更后的真实值回填到 `docs/deploy/生产上线资料收集表.md`，本文只做责任跟踪。

## 4. 资料交付规则

| 资料类型 | 允许记录 | 禁止记录 |
| --- | --- | --- |
| 密码 / Secret / API key | KMS、Vault、云密钥管理引用 | 明文值 |
| 私钥 / 证书 | 对象存储路径、证书管理系统 ID、责任人 | PEM 内容、私钥内容 |
| 管理员初始密码 | 离线交付方式、交付人、接收人、日期 | 明文密码 |
| 数据库备份 | 备份路径、校验值、恢复负责人 | 含敏感数据的下载链接公开暴露 |
| 支付资料 | 商户号、证书序列号、密钥引用 | API v3 key 明文、私钥明文 |

## 5. 阻断规则

以下任一条件未满足时，结论必须为 No-Go：

- `prod-env-check.sh` 未通过。
- 生产数据库无上线前全量备份或无法恢复。
- 生产迁移未在预生产或空生产库演练到 v24。
- 首个平台管理员资料未二人复核。
- 微信支付真实上线时，支付与退款回调未完成真实联调。
- 冒烟验证未覆盖跨租户访问边界。
- 回滚负责人、操作窗口和恢复路径未确认。
