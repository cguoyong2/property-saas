# 微信支付服务商接入方案

## 1. 目标

将当前本地 HMAC 模拟回调验签升级为微信支付 v3 服务商模式，支持：

- 平台服务商商户号；
- 租户子商户号；
- 小程序 AppID；
- 平台证书验签；
- 回调报文解密；
- nonce/timestamp 重放防护；
- 证书轮换；
- 支付、退款、对账失败告警。

## 2. 当前边界

当前后端已完成：

- 订单、订单账单关系、支付流水、退款单、退款流水分表；
- 支付/退款回调幂等；
- 回调金额校验；
- 根据订单或退款单反查租户；
- `WechatPayClient` 接入边界；
- 开发模式 `DEV_SIMULATED` 本地 HMAC 模拟验签；
- 生产模式 `REAL_WECHAT_V3` 配置门禁，未接入真实 v3 原始报文验签前不会继续走模拟验签。

当前仍未直接调用微信支付 SDK 或微信 HTTP API，不能作为生产支付实现。

## 3. 配置模型

`tenant_pay_config` 继续作为租户支付配置表。

| 字段 | 服务商模式含义 |
|---|---|
| `merchant_mode` | 固定为 `SERVICE_PROVIDER` |
| `mch_id` | 服务商商户号 |
| `sub_mch_id` | 租户子商户号 |
| `app_id` | 小程序 AppID 或服务商绑定 AppID |
| `api_v3_key_encrypted` | API v3 key 密文或密钥管理系统引用 |
| `cert_serial_no` | 商户证书序列号 |
| `cert_object_key` | 商户私钥/证书对象存储路径或 KMS 引用 |
| `notify_url` | 支付回调地址 |
| `settlement_mode` | `SERVICE_PROVIDER_SETTLE` 或租户直清算策略 |

生产环境不得在数据库中保存明文 API v3 key、私钥或证书内容。

## 4. 后端改造点

建议新增以下组件：

```text
payment/wechat/WechatPayClient
payment/wechat/WechatPayProperties
payment/wechat/WechatPayCertificateProvider
payment/wechat/WechatPayNotificationVerifier
payment/wechat/WechatPayNotificationDecryptor
payment/wechat/WechatPayReplayGuard
```

职责划分：

- `WechatPayClient`：统一封装微信支付下单、退款、查询和关单；
- `WechatPayCertificateProvider`：加载商户私钥、商户证书、平台证书和证书序列号；
- `WechatPayNotificationVerifier`：使用平台证书校验 `Wechatpay-Signature`；
- `WechatPayNotificationDecryptor`：使用 API v3 key 解密 `resource.ciphertext`；
- `WechatPayReplayGuard`：校验 `Wechatpay-Timestamp` 与 `Wechatpay-Nonce`，并写入 Redis 防重放；
- `PaymentService` / `PaymentRefundService`：只处理业务幂等、金额校验、账单核销和退款回写。

## 5. 回调验收要求

支付回调必须校验：

1. 请求头 `Wechatpay-Timestamp` 在允许时间窗口内；
2. 请求头 `Wechatpay-Nonce` 在 Redis 中未出现过；
3. 请求头 `Wechatpay-Serial` 能匹配可用平台证书；
4. `Wechatpay-Signature` 验签通过；
5. `resource` 解密成功；
6. 回调订单号可反查租户、项目和订单；
7. 回调金额与订单金额一致；
8. 同一 `transaction_id` 重复回调幂等；
9. 回调成功后只核销本租户本项目账单；
10. 验签失败、解密失败、金额不一致、订单不存在必须告警并记录接口调用日志。

退款回调同理，必须按退款单反查租户并校验退款金额。

## 6. Redis 防重放 Key

建议格式：

```text
tenant:{tenantId}:wechatpay:nonce:{serial}:{nonce}
```

TTL 建议与 `WECHAT_PAY_CALLBACK_TOLERANCE_SECONDS` 一致，默认 300 秒。

## 7. 生产环境变量

```text
WECHAT_PAY_MODE=REAL_WECHAT_V3
WECHAT_PAY_CALLBACK_TOLERANCE_SECONDS=300
PROPERTY_SAAS_JWT_SECRET=<strong-secret>
DB_URL=<jdbc-url>
DB_USERNAME=<db-user>
DB_PASSWORD=<db-password>
REDIS_HOST=<redis-host>
REDIS_PORT=<redis-port>
REDIS_PASSWORD=<redis-password>
```

商户证书、API v3 key 和私钥应通过 KMS/对象存储密钥引用读取，不建议直接通过环境变量传递完整私钥。

## 8. 禁止事项

- 禁止继续使用 `change_me_encrypted` 作为生产支付密钥；
- 禁止只校验请求体签名字段；
- 禁止从请求参数直接信任 `tenantId`；
- 禁止回调失败后直接吞掉异常；
- 禁止同一回调重复生成流水；
- 禁止平台统一收款后不记录租户/子商户清算关系。
