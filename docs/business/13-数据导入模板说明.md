# 13-数据导入模板说明

## 1. 租户开通信息表

字段：物业公司名称、统一社会信用代码、联系人、手机号、服务开始日期、服务结束日期、套餐、项目上限、账号上限、短信额度、存储额度、Logo、客服电话。

## 2. 项目/小区模板

字段：项目名称、项目编码、项目类型、省、市、区县、详细地址、负责人、联系电话、客服电话、状态。

## 3. 楼栋/单元/房屋模板

字段：项目编码、楼栋名称、单元名称、楼层、房号、建筑面积、套内面积、用途、房屋状态、收费对象、备注。

当前工程包已提供可直接导入的 CSV 模板：`docs/business/templates/house_import_template.csv`。

CSV 字段：

| 字段 | 说明 |
| --- | --- |
| `projectId` | 项目 ID，必须等于本次导入批次项目 |
| `buildingId` | 楼栋 ID |
| `unitId` | 单元 ID，可为空 |
| `houseNo` | 房号 |
| `floorNo` | 楼层 |
| `buildingArea` | 建筑面积 |
| `innerArea` | 套内面积 |
| `houseUsage` | 用途，如 `RESIDENTIAL` |
| `houseStatus` | 状态，如 `VACANT/OCCUPIED/RENTED/LOCKED/RENOVATING` |
| `chargeObject` | 收费对象，如 `HOUSE` |

## 4. 业主/住户模板

字段：项目编码、房号、姓名、手机号、证件类型、证件号、关系类型、是否户主、生效日期、失效日期。

## 5. 车辆/车位模板

字段：项目编码、房号、车牌号、车辆类型、车位编号、车位类型、月租状态、开始日期、结束日期。

## 6. 收费标准模板

字段：项目编码、收费项目、标准名称、计费方式、单价、周期、账期、到期日、滞纳金规则、生效日期、失效日期。

## 7. 历史账单模板

字段：项目编码、房号、收费项目、账期、应收金额、已收金额、欠费金额、到期日、账单状态、备注。

当前工程包已提供可直接导入的 CSV 模板：`docs/business/templates/history_bill_import_template.csv`。

CSV 字段：

| 字段 | 说明 |
| --- | --- |
| `projectId` | 项目 ID，必须等于本次导入批次项目 |
| `itemId` | 收费项目 ID |
| `standardId` | 收费标准 ID，可为空 |
| `objectType` | 账单对象类型：`HOUSE/VEHICLE/SPACE` |
| `objectId` | 账单对象 ID |
| `memberId` | 会员 ID，可为空，由对象反查补齐 |
| `houseId` | 房屋 ID，可为空，由对象反查补齐 |
| `billPeriod` | 账期，格式 `yyyy-MM` |
| `receivableAmount` | 应收金额 |
| `discountAmount` | 优惠金额 |
| `dueDate` | 到期日，格式 `yyyy-MM-dd` |

## 8. 导入要求

1. 所有导入必须先预校验，不得直接入库。
2. 错误信息必须定位到行号和字段。
3. 导入成功后生成批次号。
4. 支持按批次查询导入结果。
5. 关键业务导入需支持作废或回滚策略。
6. 所有导入批次统一写入 `import_batch`，错误明细统一写入 `import_error_detail`。
7. 导入文件本身必须先写入 `file_object`，并通过 `source_file_id` 关联到导入批次。
8. 历史账单、业主证件、车辆等敏感导入必须记录操作日志，并在错误报告中避免明文输出完整证件号。

## 9. 当前接口闭环

1. 先通过 `POST /api/files` 上传 CSV 文件，`moduleCode` 使用 `import`，记录返回的 `fileId`。
2. 房屋导入调用 `POST /api/import/house-csv`，传入 `projectId` 和 `sourceFileId`。
3. 历史账单导入调用 `POST /api/import/history-bill-csv`，传入 `projectId` 和 `sourceFileId`。
4. 导入结果通过 `GET /api/import/batches` 和 `GET /api/import/batches/{batchId}` 查询。
5. 错误明细通过 `GET /api/import/batches/{batchId}/errors` 查询，或通过 `GET /api/import/batches/{batchId}/errors.csv` 下载。
