# 14-报表口径说明

## 1. 收费报表

| 指标 | 计算口径 |
|---|---|
| 应收金额 | 查询周期内账单应收金额合计，不含已作废账单 |
| 实收金额 | 查询周期内支付成功流水金额合计 |
| 净入账金额 | 支付成功流水金额 - 退款成功流水金额 |
| 欠费金额 | 待支付、部分支付、逾期账单剩余应收金额合计 |
| 收缴率 | 净入账金额 / 应收金额 |
| 退款金额 | 退款成功流水金额合计 |

注意：应收按账期统计，实收按支付时间统计，两个口径不能混用。

## 2. 工单报表

- 工单总数；
- 待受理数；
- 处理中数；
- 超时数；
- 完成数；
- 完成率；
- 平均处理时长；
- 满意度；
- 按人员绩效统计。

当前接口实现 `totalCount`、`pendingCount`、`processingCount`、`overdueCount`、`completedCount`、`completionRate`、`averageScore`。人员绩效明细保留为后续分页明细报表。

## 3. 租赁报表

- 可租面积；
- 已租面积；
- 出租率；
- 空置资源；
- 租金应收；
- 租金实收；
- 合同到期提醒；
- 客户转化率。

租金应收按 `fee_bill.object_type = CONTRACT` 且 `source_type = LEASE_CONTRACT` 统计。

## 4. 巡检报表

- 应巡检任务数；
- 已完成任务数；
- 漏检任务数；
- 异常数；
- 整改完成数；
- 完成率；
- 漏检率。

## 5. 平台运营报表

- 租户总数；
- 活跃租户数；
- 到期租户数；
- 项目总数；
- 房屋总数；
- 后台账号数；
- 小程序用户数；
- 短信用量；
- 文件存储用量；
- 接口失败次数。

平台运营报表走平台权限，只统计全平台数据；租户经营报表必须按 `tenant_id` 和用户项目数据范围过滤。
