# 06-业务状态机

## 1. 租户状态

```text
PENDING -> TRIAL -> ACTIVE -> ARREARS_LIMITED -> SUSPENDED -> ARCHIVED
PENDING -> ACTIVE
TRIAL -> SUSPENDED
ACTIVE -> SUSPENDED
SUSPENDED -> ACTIVE
```

| 状态 | 说明 |
|---|---|
| PENDING | 待开通 |
| TRIAL | 试用中 |
| ACTIVE | 正式启用 |
| ARREARS_LIMITED | 欠费限制 |
| SUSPENDED | 暂停服务 |
| ARCHIVED | 归档 |

## 2. 房屋绑定状态

```text
PENDING -> APPROVED
PENDING -> REJECTED
APPROVED -> UNBOUND
APPROVED -> EXPIRED
REJECTED -> PENDING
```

## 3. 账单状态

```text
UNPAID -> PAYING -> PAID
UNPAID -> OVERDUE
UNPAID -> VOID
PAID -> REFUNDING -> REFUNDED
PAID -> PARTIAL_REFUNDED
PARTIAL_PAID -> PAID
PARTIAL_PAID -> VOID
```

| 状态 | 说明 |
|---|---|
| UNPAID | 待支付 |
| PAYING | 支付中 |
| PARTIAL_PAID | 部分支付 |
| PAID | 已支付 |
| OVERDUE | 已逾期 |
| VOID | 已作废 |
| REFUNDING | 退款中 |
| REFUNDED | 已退款 |
| PARTIAL_REFUNDED | 部分退款 |

## 4. 支付订单状态

```text
PENDING -> PAYING -> PAID
PENDING -> CLOSED
PAYING -> PAID
PAYING -> FAILED
PAID -> REFUNDING -> REFUNDED
PAID -> PARTIAL_REFUNDED
```

## 5. 退款状态

```text
APPLYING -> AUDIT_PASSED -> REFUNDING -> REFUNDED
APPLYING -> AUDIT_REJECTED
REFUNDING -> FAILED
FAILED -> REFUNDING
FAILED -> CLOSED
```

| 状态 | 说明 |
|---|---|
| APPLYING | 退款申请中 |
| AUDIT_PASSED | 审批通过，待提交渠道 |
| AUDIT_REJECTED | 审批拒绝 |
| REFUNDING | 渠道退款中 |
| REFUNDED | 已退款 |
| FAILED | 退款失败，可按规则重试 |
| CLOSED | 已关闭 |

## 6. 工单状态

```text
SUBMITTED -> ACCEPTED -> DISPATCHED -> PROCESSING -> WAIT_CONFIRM -> COMPLETED -> EVALUATED
SUBMITTED -> CANCELLED
ACCEPTED -> REJECTED
DISPATCHED -> TRANSFERRED
PROCESSING -> HANG_UP -> PROCESSING
WAIT_CONFIRM -> REWORK -> PROCESSING
```

| 状态 | 说明 | 处理人 |
|---|---|---|
| SUBMITTED | 已提交/待受理 | 客服/管家 |
| ACCEPTED | 已受理 | 客服/管家 |
| DISPATCHED | 已派单 | 工程/处理人 |
| PROCESSING | 处理中 | 工程/处理人 |
| HANG_UP | 挂起 | 工程/项目经理 |
| WAIT_CONFIRM | 待业主确认 | 业主/住户 |
| COMPLETED | 已完成 | 系统/客服 |
| EVALUATED | 已评价 | 业主/住户 |
| CANCELLED | 已取消 | 提交人/客服 |
| REJECTED | 已驳回 | 客服/项目经理 |

## 7. 巡检任务状态

```text
PENDING -> IN_PROGRESS -> COMPLETED
PENDING -> MISSED
IN_PROGRESS -> EXCEPTION -> RECTIFYING -> RECTIFIED -> COMPLETED
```

## 8. 合同状态

```text
DRAFT -> WAIT_EFFECTIVE -> ACTIVE -> WILL_EXPIRE -> EXPIRED
ACTIVE -> TERMINATED
DRAFT -> VOID
```

## 9. 访客状态

```text
PENDING -> APPROVED -> USED
PENDING -> REJECTED
APPROVED -> EXPIRED
APPROVED -> CANCELLED
```

## 10. 导入批次状态

```text
VALIDATING -> VALID_FAILED
VALIDATING -> IMPORTING -> SUCCESS
IMPORTING -> PARTIAL_SUCCESS
IMPORTING -> FAILED
SUCCESS -> VOIDED
PARTIAL_SUCCESS -> VOIDED
```

## 11. 状态机开发要求

1. 状态不能只在前端控制，后端必须校验当前状态是否允许流转。
2. 每次状态流转必须记录操作人、时间、原因、前状态、后状态。
3. 高风险状态流转必须记录审计日志。
4. 状态枚举不得随意新增；需要新增时必须同步数据库、接口、前端、测试用例。
5. OpenAPI 中的状态枚举、数据库字段注释和本文件必须保持一致。
