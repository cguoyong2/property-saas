# 00-CODEX使用说明

## 0. 首次使用强制流程

第一次把工程包交给 Codex 时，不允许直接开发，不允许执行 `TASK-001`。

必须先复制并执行：

```text
prompts/Codex首次只读预审提示词.md
```

只有当 Codex 输出《工程包预审问题清单》，并且严重问题已处理或确认不存在后，才允许进入正式开发任务。

## 1. 推荐工作方式

不要把整个系统一次性丢给 Codex。正确做法是：

1. 初始化仓库结构；
2. 提交本工程包；
3. 若工程包仍是 zip，先解压到仓库根目录，或明确让 Codex 从 zip 内读取目标文件；
4. 让 Codex 先执行 `prompts/Codex首次只读预审提示词.md`，只读预审，不改文件；
5. 预审问题处理后，让 Codex 阅读 `AGENTS.md`、`docs/00-开发前冻结事项.md`；
6. 按 `tasks/TASK-xxx` 一个任务一个任务执行；
7. 每个任务完成后要求 Codex 输出：改动文件、实现内容、测试结果、未完成事项；
8. 项目经理/技术负责人审核后再进入下一个任务。

## 2. 每次给 Codex 的标准提示词

参考 `prompts/Codex单任务开发提示词模板.md`。

## 3. 不推荐的做法

- 不要说“请把整个物业系统开发出来”。
- 不要让 Codex 自己设计数据库。
- 不要让 Codex 自己决定支付模式。
- 不要让 Codex 自己决定权限模型。
- 不要让 Codex 自己补接口路径。
- 不要让 Codex 在没有验收用例的情况下声明完成。

## 4. 已冻结事项与仍需确认事项

已冻结事项详见 `docs/00-开发前冻结事项.md`：

1. 技术版本已冻结；
2. 支付模型按服务商模式设计，兼容租户自有商户号；
3. 平台运营端与物业管理端共用同一个 PC 管理端项目；
4. 微信端使用 uni-app 微信小程序；
5. 门禁、梯控、停车、监控第一版只做适配层，不深度绑定厂商；
6. 首次交给 Codex 必须只读预审。

仍需业务方后续确认的事项：

1. SaaS 商业收费规则：按租户、按项目、按户数、按功能套餐还是混合收费；
2. 微信支付真实商户资料、服务商资料、子商户进件流程；
3. 小程序主体、名称、AppID、微信认证主体；
4. 短信、门禁、梯控、停车等第三方接口文档和测试账号；
5. 是否包含旧系统历史数据迁移、清洗、导入和校验。

## 5. 代码审查重点

- 任意业务查询是否带租户隔离。
- 任意支付/退款回调是否幂等。
- 任意高风险操作是否有日志。
- 任意金额计算是否使用 BigDecimal。
- 任意导出是否有权限与数据范围控制。
- 任意第三方接口是否有失败重试与日志。
