# v1.1 更新说明

## v1.1.1 预审修订

本次根据首次只读预审结果，修订工程包内部契约不一致问题：

1. 补齐 `tenant_pay_config` 支付配置表，明确服务商模式和普通商户模式所需字段。
2. 将 `member_user` 调整为租户会员档案表，补充 `tenant_id` 与租户内唯一约束。
3. 补齐 `pay_refund_transaction`、`import_batch`、`import_error_detail`、第三方调用重试字段。
4. 扩展 `openapi/openapi.yaml`，补齐 PC 管理端和小程序页面清单引用的接口。
5. 统一 OpenAPI 响应、错误响应、状态枚举和主要请求体。
6. 同步权限矩阵、状态机、数据库设计说明、第三方接口边界、数据导入说明和验收用例映射。
7. 补充 README/使用说明中关于 zip 解压或 zip 内读取的说明。
8. 人工确认并冻结两项口径：`member_user` 采用租户内会员档案模型；支付优先服务商/子商户模式并兼容租户自有普通商户号，不默认平台统一收款。

## 本次冻结事项

1. 技术版本冻结：Java 17、Spring Boot 3.2.x、MySQL 8.0、Redis 7、Vue3、uni-app + Vue3；
2. 支付模式冻结：按微信支付服务商模式设计，兼容租户自有普通商户号，不默认采用平台统一收款；
3. 前端边界冻结：平台运营端与物业管理端共用一个 `admin-web`，微信端为 `mobile-uniapp`；
4. 设备接口冻结：门禁、梯控、停车、监控第一版只做适配层和接口预留，不写死厂商 SDK；
5. Codex 首次任务冻结：第一次必须执行只读预审，不允许写代码、不允许修改文件、不允许执行 `TASK-001`。
